## O que são Ferramentas de IA?

Ferramentas são **ações que o agente pode executar** durante uma conversa. Pense nelas como "superpoderes" que você dá ao seu agente para ele fazer mais do que só conversar.

<Info>
  Enquanto a Base de Conhecimento ensina o agente a **saber**, as Ferramentas ensinam o agente a **fazer**.
</Info>

    ```mermaid
    sequenceDiagram
        Cliente->>Agente: "Consultar estoque do produto X"

        Seu Sistema->>Sua API: Retorna dados
        Sua API->>Integrabot: JSON com resultado
        Integrabot->>Agente: Processa resposta
        Agente->>Cliente: "Temos 15 unidades disponíveis"
    ```

## Criar Ferramenta Customizada

### Passo a Passo

<Steps>
  <Step title="Desenvolva sua API">
    Crie um endpoint que:
    - Aceita requisição POST com JSON
    - Processa a ação necessária
    - Retorna JSON com resultado
    
    **Exemplo em Node.js:**
    ```javascript
    app.post('/api/consultar-estoque', async (req, res) => {
      const { produto_id } = req.body;
      
      // Busca no seu sistema
      const estoque = await buscarEstoque(produto_id);
      
      // Retorna resultado
      res.json({
        disponivel: estoque > 0,
        quantidade: estoque,
        mensagem: `Temos ${estoque} unidades disponíveis`
      });
    });
    ```
  </Step>
  
  <Step title="Acesse Ferramentas IA">
    No Integrabot:
    1. Menu lateral → **Ferramentas IA**
    2. Clique em **+ Nova Ferramenta**
  </Step>
  
  <Step title="Configure a Ferramenta">
    Preencha:
    
    **Nome:** `consultar_estoque`
    (Sem espaços, minúsculo, use underscore)
    
    **Descrição:**
    ```
    Consulta a quantidade disponível em estoque de um produto.
    Use quando o cliente perguntar sobre disponibilidade.
    ```
    
    **Endpoint:** `https://sua-api.com/api/consultar-estoque`
    
    **Método:** POST
    
    **Cabeçalhos (Headers):**
    ```json
    {
      "Authorization": "Bearer SEU_TOKEN_AQUI",
      "Content-Type": "application/json"
    }
    ```
    
    **Parâmetros:**
    ```json
    {
      "type": "object",
      "properties": {
        "produto_id": {
          "type": "string",
          "description": "ID do produto a consultar"
        }
      },
      "required": ["produto_id"]
    }
    ```
  </Step>
  
  <Step title="Teste">
    Use o botão **Testar** para fazer uma chamada de teste
    
    Verifique se o retorno está correto
  </Step>
  
  <Step title="Ative">
    Clique em **Salvar e Ativar**
    
    A ferramenta estará disponível para o agente!
  </Step>
  
  <Step title="Configure no Agente">
    Vá em **Meu Agente** → **Módulos e Ferramentas**
    
    Certifique-se que a ferramenta está ✅ ativada
    
    Adicione nas instruções:
    ```
    Quando o cliente perguntar sobre disponibilidade de produto,
    use a ferramenta consultar_estoque para verificar o estoque
    em tempo real.
    ```
  </Step>
</Steps>

<Check>
  Pronto! Seu agente agora pode consultar estoque em tempo real.
</Check>

## Estrutura de Parâmetros

Parâmetros seguem o padrão **JSON Schema**:

<Accordion title="Tipos de Dados Suportados">
  ### String (Texto)
  ```json
  {
    "nome_cliente": {
      "type": "string",
      "description": "Nome completo do cliente"
    }
  }
  ```
  
  ### Number (Número)
  ```json
  {
    "quantidade": {
      "type": "number",
      "description": "Quantidade desejada"
    }
  }
  ```
  
  ### Boolean (Sim/Não)
  ```json
  {
    "urgente": {
      "type": "boolean",
      "description": "Se é urgente ou não"
    }
  }
  ```
  
  ### Array (Lista)
  ```json
  {
    "produtos": {
      "type": "array",
      "items": {
        "type": "string"
      },
      "description": "Lista de IDs de produtos"
    }
  }
  ```
  
  ### Object (Objeto)
  ```json
  {
    "endereco": {
      "type": "object",
      "properties": {
        "rua": { "type": "string" },
        "numero": { "type": "string" },
        "cep": { "type": "string" }
      },
      "description": "Endereço completo"
    }
  }
  ```
</Accordion>

<Accordion title="Validações e Restrições">
  ### Enum (Opções limitadas)
  ```json
  {
    "estado": {
      "type": "string",
      "enum": ["SP", "RJ", "MG", "PR"],
      "description": "Estado (apenas sudeste)"
    }
  }
  ```
  
  ### Min/Max para Números
  ```json
  {
    "quantidade": {
      "type": "number",
      "minimum": 1,
      "maximum": 100,
      "description": "Quantidade (1-100)"
    }
  }
  ```
  
  ### Pattern (Regex)
  ```json
  {
    "cep": {
      "type": "string",
      "pattern": "^[0-9]{5}-?[0-9]{3}$",
      "description": "CEP no formato 12345-678"
    }
  }
  ```
</Accordion>

## Formato de Resposta

Sua API deve retornar JSON. O agente processará e usará na resposta.

<Tabs>
  <Tab title="Resposta Simples">
    ```json
    {
      "sucesso": true,
      "mensagem": "Produto disponível em estoque",
      "dados": {
        "quantidade": 15,
        "preco": 99.90
      }
    }
    ```
    
    O agente pode usar qualquer campo na resposta ao cliente.
  </Tab>
  
  <Tab title="Resposta com Erro">
    ```json
    {
      "sucesso": false,
      "erro": "Produto não encontrado",
      "codigo": "PRODUTO_NAO_ENCONTRADO"
    }
    ```
    
    O agente tratará o erro e informará o cliente de forma adequada.
  </Tab>
  
  <Tab title="Resposta Estruturada">
    ```json
    {
      "disponivel": true,
      "produto": {
        "id": "123",
        "nome": "Notebook Dell",
        "preco": 3500.00,
        "estoque": 5,
        "prazo_entrega": "3-5 dias úteis"
      },
      "mensagem_cliente": "Produto disponível com entrega rápida!"
    }
    ```
    
    Quanto mais estruturado, melhor o agente pode formular a resposta.
  </Tab>
</Tabs>

<Tip>
  **Dica:** Inclua um campo `mensagem_cliente` ou similar que o agente possa usar diretamente na resposta.
</Tip>

## Segurança

<Warning>
  **Importante:** Sempre proteja suas APIs!
</Warning>

<CardGroup cols={2}>
  <Card title="Use Autenticação" icon="key">
    - API Key no header
    - Bearer Token
    - OAuth 2.0
    
    Nunca deixe endpoints públicos sem autenticação
  </Card>
  
  <Card title="Valide Entrada" icon="shield-check">
    - Valide todos os parâmetros
    - Sanitize inputs
    - Limite rate (requisições por minuto)
  </Card>
  
  <Card title="Use HTTPS" icon="lock">
    - Sempre SSL/TLS
    - Nunca HTTP puro
    - Certificado válido
  </Card>
  <Card title="Logs e Monitoramento" icon="chart-line">
    - Registre todas as chamadas
    - Monitore erros
    - Alerte em falhas
  </Card>
</CardGroup>

## Exemplos Práticos

<AccordionGroup>
  <Accordion title="Calcular Frete por CEP" icon="truck">
    **Endpoint:** `POST /api/calcular-frete`
    
    **Parâmetros:**
    ```json
    {
      "type": "object",
      "properties": {
        "cep_destino": {
          "type": "string",
          "description": "CEP de entrega"
        },
        "peso": {
          "type": "number",
          "description": "Peso em kg"
        }
      },
      "required": ["cep_destino"]
    }
    ```
    
    **Resposta:**
    ```json
    {
      "valor": 25.90,
      "prazo": "5-7 dias úteis",
      "transportadora": "Correios - PAC"
    }
    ```
    
    **Instruções do Agente:**
    ```
    Quando o cliente perguntar sobre frete, use a ferramenta
    calcular_frete com o CEP informado. Apresente o valor e
    prazo de forma clara.
    ```
  </Accordion>
  
  <Accordion title="Validar Cupom de Desconto" icon="ticket">
    **Endpoint:** `POST /api/validar-cupom`
    
    **Parâmetros:**
    ```json
    {
      "type": "object",
      "properties": {
        "codigo_cupom": {
          "type": "string",
          "description": "Código do cupom"
        },
        "valor_compra": {
          "type": "number",
          "description": "Valor total da compra"
        }
      },
      "required": ["codigo_cupom"]
    }
    ```
    
    **Resposta:**
    ```json
    {
      "valido": true,
      "desconto_percentual": 10,
      "valor_desconto": 50.00,
      "mensagem": "Cupom válido! 10% de desconto aplicado."
    }
    ```
  </Accordion>
  
  <Accordion title="Criar Pedido" icon="shopping-cart">
    **Endpoint:** `POST /api/criar-pedido`
    
    **Parâmetros:**
    ```json
    {
      "type": "object",
      "properties": {
        "cliente_id": {
          "type": "string"
        },
        "produtos": {
          "type": "array",
          "items": {
            "type": "object",
            "properties": {
              "produto_id": { "type": "string" },
              "quantidade": { "type": "number" }
            }
          }
        },
        "endereco_entrega": {
          "type": "object",
          "properties": {
            "rua": { "type": "string" },
            "numero": { "type": "string" },
            "cep": { "type": "string" }
          }
        }
      },
      "required": ["produtos", "endereco_entrega"]
    }
    ```
    
    **Resposta:**
    ```json
    {
      "pedido_id": "PED-12345",
      "valor_total": 150.00,
      "prazo_entrega": "3-5 dias úteis",
      "link_pagamento": "https://pay.com/12345"
    }
    ```
  </Accordion>
</AccordionGroup>

## Debugging e Testes

### Testar Ferramenta

<Steps>
  <Step title="Acesse a Ferramenta">
    Em **Ferramentas IA**, clique na ferramenta
  </Step>
  
  <Step title="Clique em Testar">
    Botão "Testar" abre modal de teste
  </Step>
  
  <Step title="Preencha Parâmetros">
    Informe valores de teste para cada parâmetro
  </Step>
  
  <Step title="Execute">
    Clique em "Executar Teste"
    
    Veja a requisição, resposta e tempo
  </Step>
</Steps>

### Logs de Execução

Todas as chamadas de ferramentas são logadas:

1. Acesse **Ferramentas IA**
2. Clique na ferramenta
3. Aba **Logs**

Você verá:
- Timestamp
- Parâmetros enviados
- Resposta recebida
- Tempo de execução
- Status (sucesso/erro)

<Tip>
  Use logs para debugar problemas e entender como o agente está usando a ferramenta.
</Tip>

## Limitações

<Warning>
  **Limites de uso:**
  - Timeout: 30 segundos por chamada
  - Tamanho de resposta: 100 KB
  - Chamadas simultâneas: 10
  - Rate limit: 100 requisições/minuto por ferramenta
</Warning>
